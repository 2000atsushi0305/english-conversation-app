## 1. プロダクト概要

### サービス名
**boba Talk**（コードベース上は `TalkBoba`）

### 一言で言うと
**「お気に入りの ボバちゃん と毎日5分、英語でゆるくおしゃべりする」AI英会話練習アプリ**

### 提供価値
- いつでも・何度でも・恥ずかしがらずに英語を「話す」練習ができる
- 自分の英語が「合ってるか」その場で評価してもらえる安心感
- キャラ・声・会話の温度感が居心地よくて、続けたくなる
- 月¥480 で、語学コーチを毎日ポケットに入れているのと同等

### URL / アクセス手段
- 本番: https://justconvo.net （PWA対応、スマホでホーム画面追加可）
- 開発: http://localhost:8080

## 2. プロダクト原則 ⭐

**boba Talk が「やる/やらない」を即決するための6つの原則。** どんな機能追加・改善も、これらに照らして判断する。

### 原則1: 話す量を最優先する
ユーザーが「英語を口から出す回数」を最大化する。何かを足すなら、これを増やすか維持するかが基準。

- ✅ 採用例: ヒントボタン（言葉に詰まらず会話続行）、練習ボタン（直された文を即言い直し）
- ❌ 却下例: TOEIC 文法ドリル、単語フラッシュカード（話す量を増やさない）

### 原則2: 学習効果と「楽しさ」を両立する
どちらか一方では続かない。両立するから差別化される。

- ✅ 採用例: 6キャラ × 自然な声 + その場で文法添削
- ❌ 却下例: 可愛いだけのキャラ会話（学習効果薄い）、ガチ英文法解説サイト（楽しくない）

### 原則3: ユーザーを否定しない
間違いを指摘する時も、Bobaは「おしい！」「ちっちゃい直しだけ」のような肯定的トーン。心理的安全性が継続率の鍵。

- ✅ 採用例: 評価チップに ❌ や「間違い」の文字を使わない
- ❌ 却下例: スコア低下表示、赤字で減点表示、「不正解」テキスト

### 原則4: 難しい英語より「伝わる英語」
ネイティブ完璧主義より、ユーザーが現実に使える英語を優先。CEFR A2〜B1 中心。

- ✅ 採用例: 「I went to Osaka.」のような実用文を Boba が使う
- ❌ 却下例: 高度な口語イディオム連発、SAT/英検1級レベルの語彙

### 原則5: 毎日5分続けられる設計
1回のセッションは短く、毎日入り口が軽い。長時間プレイを促す仕掛けは作らない。

- ✅ 採用例: 「今日の一言3往復」のクイックモード、ストリーク🔥
- ❌ 却下例: 30分必須レッスン、長時間で増えるボーナス、依存的なゲーム性

### 原則6: 安価で高品質を維持する
¥480 という価格は絶対に守る。コスト効率の悪い機能は採用しない。

- ✅ 採用例: TTSキャッシュで API コスト圧縮、Gemini Flash でチャットコスト最小化
- ❌ 却下例: ElevenLabs などプレミアム TTS（粗利が崩れる）、ライブ通訳機能

### 適用ルール
新機能を検討する時、**必ずこの6原則のチェックリストを通す**こと。
3つ以上に「✅貢献する」と言えなければ却下、または再設計。

### メインペルソナ
**「中学英語は分かるけど話せない」社会人 20〜40代**

| 属性 | 詳細 |
|------|------|
| 年齢 | 25〜45歳 |
| 性別 | 男女問わず |
| 英語レベル | TOEIC 400〜700、CEFR A2〜B1 |

| ライフスタイル | スマホ常用、通勤や寝る前など隙間時間に勉強したい |

### ペルソナの本音
- 「英会話スクールは高い・人前で間違えるのが恥ずかしい」
- 「DMM英会話・ネイティブキャンプは予約が面倒・先生選びに疲れる」
- 「アプリ系（Duolingo等）は単語ばっかりで実際に話す機会がない」
- 「自分の英語が通じてるか不安・直してくれる人がいない」
- 「キャラ系アプリは可愛いけど学習効果が薄そう」

### 既存解決手段との比較
| 競合 | 価格 | 強み | 弱み（boba Talk が刺さる点） |
|------|------|------|---------------------------|
| DMM英会話 | ¥7,000〜/月 | ネイティブ講師 | 予約面倒・人前で恥ずかしい |

| Duolingo | 無料〜¥1,300/月 | ゲーム感覚 | 話す練習ほぼなし |
| Speak (AI) | $20/月 | 米系AI英会話 | 英語UI・日本人向けじゃない |
| ELSA Speak | $11.99/月 | 発音特化 | 会話練習ではない |
| **boba Talk** | **¥480/月** | **キャラと話せる + 自然な声 + その場で添削** | （対応中） |

### 差別化ポイント
1. **6種類のキャラから選べる**: 性格 × 声 × 得意テーマ
2. **「気軽に話せる相手」が居る**: 緊張感ゼロ、ノンジャッジ
3. **その場で添削＋自然な言い方提案**: 自分の英語が「合ってる/もっと良い」がすぐ分かる
4. **苦手分野が可視化**: 同じミスを繰り返してることに気付ける
5. **月¥480**: 競合の1/10〜1/15 の価格

## 4. 解決する課題

### 主課題
> **「英語を話したいけど、恥ずかしくて練習できない & 自分の英語が合ってるか分からない」**

### 副次的課題
- 英会話スクールに通うほどの予算/時間がない
- 続けるモチベが湧かない
- 何を話したらいいか分からない（会話が続かない）
- 同じミスを何度もしていることに気付けない
- AI英会話は「面白くない」「機械的」で続かない

## 5. 機能要件

### 5.1 コア機能（毎日使う）

#### A. AI との英会話
- テキスト入力 / 音声入力（マイク）両対応
- AI（Groq Llama 3.3 70B）が文脈を理解して英語で応答
- 各キャラの **persona** に従って口調・話題傾向が変わる
- 日本語訳が常に併記される（理解の補助）

#### B. その場のフィードバック（評価チップ）
ユーザーの発話に対して**毎回 5段階評価**を返す（折りたたみ式UI）:
- ✨ Perfect — 完璧
- 👍 Natural — 自然
- 💡 通じる — 文法OK、もっと自然な言い方アリ → 候補1〜3個提示
- 📝 [ミスタグ] — 文法ミスあり → ルール + 例文 + 練習ボタン
- ❓ うまく取れず — 意味取れない → リトライ促し

#### C. キャラクター選択
- 6キャラから自由選択（無料: 2 / 有料: 4）
- いつでも切替可（ドロワー設定で 1tap）
- キャラごとに専用の **Chirp 3 HD ボイス**
- ロック中キャラ tap → プラン誘導

| キャラ | 性別 | 得意 | ボイス | プラン |
|--------|------|------|------|------|
| 🫧 ミルクボバ | 女性 | 日常会話 | Aoede | 無料 |
| 🍵 抹茶ボバ | 女性 | ビジネス | Kore | 無料 |
| 🔥 黒糖ボバ | 男性 | 趣味 | Charon | 有料 |
| 🍓 いちごボバ | 子供 | 食べ物 | Puck | 有料 |
| ☕ コーヒーボバ | 男性 | 映画音楽 | Fenrir | 有料 |
| 🌸 桜ボバ | 女性 | 旅行 | Zephyr | 有料 |

#### D. テーマ選択
6テーマから1tapで切替: 日常 / 旅行 / ビジネス / 趣味 / 食べ物 / 映画音楽
各テーマに「得意キャラ」のサジェスト表示。

#### E. 自然な発音の音声再生
- AI返答が **Google Cloud Chirp 3 HD** で音声化
- AudioContext で自動再生（自動再生制限を回避）
- 同じ文は DB キャッシュで即時再生（173倍速）

### 5.2 学習補助機能

#### F. 返し方ヒント（💡ボタン）
何を言えばいいか分からない時、AI が文脈に合った返答候補を3つ提示。

#### G. 練習ボタン
評価チップ展開時に「🎯この英文で1回練習する」ボタン。入力欄に正しい英文がプリセットされる。

#### H. 苦手分野 TOP（Stats モーダル）
直近30日のミスを集計して、🥇🥈🥉で苦手TOP3を表示。
カテゴリ: 時制 / 冠詞 / 前置詞 / 語順 / 三人称単数の一致 etc.

#### I. 今日の一言チャレンジ
日替わりで実用フレーズ1つを提示。クイックモードで3往復練習。

### 5.3 継続支援機能

#### J. ストリーク（🔥連続日数）
連続でアプリ使った日数を表示・カウントアップ。
3/7/14/30/50/100日のマイルストーンで紙吹雪 + 祝祭バナー演出。

#### K. テーマ別レベル成長
6テーマそれぞれにレベル（初心者→慣れてきた→中級者→上級者→エキスパート→🏆マスター）。
プログレスバーで成長可視化。

#### L. 振り返り（レビューモーダル）
2往復以上した後の新会話ボタンで、その会話で出た有用表現を抽出・カード表示。
「次回チャレンジ」として表現を保存して、次回会話開始時に提案。

#### M. 毎日のリマインダー（📬 Push通知）
ONにすると毎日「今日の一言」が通知で届く（PWAインストール時）。

### 5.4 ユーザー機能

#### N. アカウント
- メールアドレス + パスワード で新規登録/ログイン
- セッション維持（Cookie）

#### O. ニックネーム設定
Bobaがニックネームで呼びかけてくれる（音声認識でも考慮）。

#### P. 紹介リンク（+70回ボーナス）
固有の紹介コードを発行。
友達が登録すると、両者に+70回の会話ボーナス。

#### Q. 会話履歴
過去5回までの会話を保存（localStorage）。
「続きから」で再開可能。

## 6. AI応答ポリシー ⭐

`system_prompt` の暗黙ルールを明文化。新キャラ追加 / モデル変更時の判断基準であり、運営時の安全網。

### 6.1 必須事項（MUST）

| 項目 | ルール |
|------|--------|
| 言語 | 英語で返答、日本語訳を必ず併記 |
| レベル | CEFR A2〜B1 を基本、難易度設定に応じて調整 |
| 長さ | 1〜3文（中級）/ 2〜4文（上級）。長文講義禁止 |
| 構造 | 会話を続ける follow-up question を末尾に含める |
| 人格 | 選択中キャラの persona を維持 |
| 添削 | naturalness を必ず出力。文法ミスは correction を省略しない |
| トーン | 否定語禁止、優しく肯定的に（原則3） |
| 日本語 | 中国語の漢字（簡体字・繁体字）混入禁止 |
| JSON | フォーマット厳守、追加テキストなし |

### 6.2 禁止事項（MUST NOT）

| カテゴリ | 例 |
|---------|-----|
| 医療助言 | 病気の診断、薬の推奨、治療法の指示 |
| 金融助言 | 投資推奨、税務アドバイス、信用情報の判断 |
| 法務助言 | 契約解釈、訴訟手続きのガイド |
| 政治・宗教 | 特定政党/宗教の擁護・批判、論争的トピックでの主張 |
| 差別 | 人種・性別・性的指向・宗教・国籍による偏見表現 |
| 性的表現 | 露骨な性的描写、性的サービスの示唆 |
| 暴力 | 暴力行為の誘発、武器の使い方説明 |
| 自殺・自傷 | 自殺・自傷の方法、否定的な強化 |
| 違法行為 | 違法行為の指南、危険物の作り方 |
| 個人情報の収集 | 住所・電話番号・口座番号などを聞き出す行為 |
| 過度の自己開示誘導 | 健康状態・家族問題等の私的詳細の聞き出し |
| 恋愛シミュレーション化 | 恋愛感情の表現・ロールプレイの強化 |
| 過度な依存促進 | 「会えるのを楽しみにしてた」等の依存誘発フレーズ |

### 6.3 グレーゾーンの扱い

- ユーザーが上記禁止トピックを話題に出した場合 → 軽くいなして話題転換（例：「Hmm, that's a heavy topic. Let's talk about something lighter — what did you eat today?」）
- 健康・お金・恋愛の相談を持ちかけられた → 「I'm just here to chat in English, not give serious advice 🫧 But how about we practice a similar phrase?」のように学習に戻す

### 6.4 インシデント対応

問題発言が観測された場合の手順：
1. ユーザーから報告（フィードバックフォーム or サポートメール）
2. 当該会話を `mistake_log` 同様にログ化（要実装：今は会話履歴がlocalStorageのみで再現困難）
3. system_prompt に「このパターンは絶対避ける」を追記
4. 必要なら緊急デプロイ
5. 利用規約に対応条項追加

## 7. ユーザー状態遷移 ⭐

### 7.1 状態定義

| 状態 | 説明 | 使える機能 |
|------|------|----------|
| **Guest** | 未登録（LP閲覧のみ） | LP・料金ページ閲覧、登録 |
| **Free** | 無料プラン、当月利用上限内（≤30回） | 全機能、ミルク+抹茶キャラ、ブラウザTTS |
| **Free_exhausted** | 無料、当月上限到達 | 履歴閲覧のみ、新規会話不可、アップグレード誘導表示 |
| **Standard** | ¥480 課金中、当月600回内 | 全機能、全6キャラ、Chirp 3 HD |
| **Standard_exhausted** | Standard、当月600回到達 | 履歴閲覧のみ、Premium 誘導 |
| **Premium** | ¥980 課金中（公正利用内） | 全機能、無制限、Chirp 3 HD |
| **Past_due** | Stripe 決済失敗中（猶予期間） | 当面 Free 相当の機能、決済更新誘導 |
| **Suspended** | 規約違反・不正利用で BAN | ログイン不可、サポート連絡先のみ表示 |
| **Deleted** | 退会済み（30日経過で完全削除） | （ログイン不可） |

### 7.2 状態遷移図

```
Guest
  └─[登録]──> Free
                ├─[30回利用]──> Free_exhausted
                │                  ├─[月初リセット]──> Free
                │                  └─[Stripe決済]───> Standard or Premium
                ├─[Stripe決済]────> Standard or Premium
                └─[退会]──────────> Deleted

Standard
  ├─[600回利用]────> Standard_exhausted
  │                   ├─[月初リセット]──> Standard
  │                   └─[Premium決済]───> Premium
  ├─[Premium変更]──> Premium
  ├─[解約]────────> Free（次回更新日まで Standard 維持）
  ├─[決済失敗]────> Past_due
  └─[退会]────────> Deleted

Premium
  ├─[Standard変更]─> Standard
  ├─[解約]────────> Free
  └─[決済失敗]────> Past_due

Past_due
  ├─[決済成功]────> Standard or Premium に復帰
  └─[7日経過]────> Free に降格 + 警告通知

Any
  └─[規約違反]────> Suspended
                      └─[解除]──> 元の状態
```

### 7.3 状態と DB の関係

`users.plan` カラムの値:
- `'free'` → Free / Free_exhausted（usage_count で判別）
- `'light'` → Standard / Standard_exhausted（usage_count で判別）
- `'premium'` → Premium
- `'past_due'` → Past_due（**現状未実装**、追加要）
- `'suspended'` → Suspended（**現状未実装**、追加要）

→ TODO: `past_due` / `suspended` 状態のサポート追加。Stripe Webhook の `invoice.payment_failed` イベント連携も。

## 8. 非機能要件

### 8.1 パフォーマンス
- AI応答: 平均1〜3秒
- TTS生成: 初回 ~1秒、キャッシュ ~6ms
- 体感遅延: AI応答開始から音声再生まで 300ms以内目標

### 8.2 対応プラットフォーム
- iOS Safari (PWA インストール推奨)
- Android Chrome
- PC Chrome / Edge / Safari
- マイク入力は Chrome/Edge 推奨（Safari もOK だが Whisper 経由でカバー）

### 8.3 セキュリティ
- HTTPS 必須（本番）
- パスワード bcrypt ハッシュ
- ログイン試行レート制限（IP単位）
- Stripe Webhook 署名検証
- API キーは .env で管理、リポジトリには含めない

### 8.4 アクセシビリティ
- 全てのインタラクティブ要素に aria 属性
- フォントサイズ 16px 以上（iOS自動ズーム対策）
- カラーコントラスト AA レベル準拠
- キーボード操作（ESC でモーダル/ドロワー閉じる等）

### 8.5 国際化
- 現状: 日本語UI、英語学習特化
- 将来: 英語UI（米国市場向け）、中国語UI などは未対応

## 9. データ保存方針 ⭐

### 9.1 保存データと保存期間

| データ種類 | 保存場所 | 保存期間 | 削除トリガー | 備考 |
|----------|---------|---------|------------|------|
| 認証情報（メアド・bcryptハッシュ） | `users` テーブル | 退会後30日 | 退会リクエスト → 30日経過で物理削除 | 30日間は誤操作復旧用 |
| プラン・利用回数 | `users` テーブル | 退会後30日 | 同上 | |
| ストリーク・成長記録 | `users`, `theme_progress` | 退会後30日 | 同上 | |
| 文法ミスログ | `mistake_log` | 90日 | 90日経過で自動削除（定期バッチ要実装） | 苦手TOP3集計用、90日で十分 |
| TTS音声キャッシュ | `tts_cache` | 永続（hit_count = 0 で180日経過分は掃除可） | LRU風削除（実装予定） | ユーザー紐付けなし、安全 |
| 会話履歴 | ブラウザ `localStorage` のみ | ユーザーが消すまで | ブラウザストレージ削除 | **サーバーには一切保存しない** |
| 紹介コード | `users.referral_code` | 退会後30日 | アカウント削除 | |
| Push購読情報 | `push_subscriptions` | 解除時即削除 | unsubscribe / 退会 | |
| アクセス解析 | `page_views` | 180日 | 180日経過で集計後に元データ削除 | 匿名（IPは保存しない） |
| フィードバック | `feedbacks` | 永続（匿名化済み） | 個別削除リクエストで対応 | user_id 紐付きだが分析用は集計のみ |

### 9.2 削除リクエスト対応

ユーザーから削除リクエストがあった場合:
1. 退会フォーム / 問い合わせから受付
2. 即座に `users` の状態を Deleted に変更（ログイン不可）
3. 30日後にバッチで物理削除（`users` および外部キー連動データ）
4. `tts_cache` `feedbacks` `page_views` は匿名なので残す

### 9.3 GDPR / 個人情報保護法 観点
- データエクスポート機能: 未実装（要対応、特にEU圏展開時）
- データ削除リクエストへの対応: 上記手順で可能
- Cookie 同意バナー: 必要に応じて追加（現状なし）

### 9.4 暗号化方針
- 通信: HTTPS（TLS 1.2+）
- パスワード: bcrypt（cost factor 12）
- DB全体暗号化: Fly.io の Volume 暗号化に依存（現状）
- 機密データ（決済情報）: Stripeに委託、自前保存しない

## 12. 成功指標 (KPI) — WSU を北極星に ⭐

### 12.1 北極星指標 (North Star Metric)

**WSU (Weekly Speaking Users) = 「週に5分以上、英語を話したユーザー数」**

なぜこれか：
- 学習成果に直結（話す = 上達）
- 継続率に直結（毎週話す習慣＝ストリーク）
- 課金率に直結（よく話すユーザーが課金する）
- キャラ改善にも直結（話したくなるキャラが伸びる）

「MAU」や「セッション数」では「ログインしただけ」を拾ってしまうのでダメ。

### 12.2 サブ KPI（AARRRファネル）

| ステージ | 指標 | 初期目標（月） |
|---------|------|------------|
| **Acquisition** | 新規登録数 | 50（初期）→ 500（半年後） |
| **Activation** | 初日5往復達成率 | 60% |
| **Retention** | 7日継続率（D7） | 25% |
| **Retention** | 30日継続率（D30） | 10% |
| **Revenue** | 無料→有料コンバージョン率 | 5〜10% |
| **Referral** | 紹介リンク経由登録率 | 15% |

### 12.3 想定ファネル例

```
登録 100人
  ↓ 初回会話 70%
   70人
  ↓ 翌日利用 57%
   40人
  ↓ 7日継続 50%
   20人
  ↓ 課金 20%
    4人
```

→ 月50登録なら月2人課金。100登録で4人。500登録で20人 ≈ ¥9,600/月。

### 12.4 補助 KPI

| KPI | 目標 |
|-----|------|
| 平均ストリーク日数 | 7日以上 |
| 月間粗利 | ¥30,000（25ユーザー有料）が初期目標 |
| 解約率（チャーン） | 月10%以下 |
| NPS（10段階で7以上の割合） | 60%以上 |
| キャラ利用分布（特定キャラに偏ってないか） | 各キャラ 5〜25% の範囲 |

| 文法ドリル | 会話の中で身につける派（原則1） |
| 単語帳機能 | フォーカス分散（原則1） |

## 14. 制約条件 / 既知の課題

### 14.1 技術的制約
- iOS Safari の自動再生制限 → AudioContext で回避済み
- ブラウザTTS（フォールバック）はロボット声、無料プランの体験が悪い

- マーケ予算: ほぼゼロ、SNS + 紹介リンクが頼り
- カスタマーサポート: 個人対応、スケール時に課題

### 14.3 機能的な技術的負債
- `past_due` / `suspended` プラン状態の判定ロジック未実装
- `mistake_log` の90日自動削除バッチ未実装
