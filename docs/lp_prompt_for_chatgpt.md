# 文体ガイドライン

- 「ですます調」基本、たまにフランクに崩す
- 専門用語は避ける（「LLM」「Web Speech API」等は NG）
- 「あなた」より「みんな」「私たち」を使うとあたたかい
- 数字は具体的に（「たくさん」より「月600回」）
- Boba のセリフを所々挿入すると親しみ UP（例：「ゆっくりでいいよ〜🫧」）

# 出力フォーマット

- **HTML 1ファイル**（CSS は `<style>` 内に内蔵、JavaScript は最小限）
- レスポンシブ対応（PC + SP の両方できれいに）
- 画像は SVG または絵文字 or CSS gradient で表現（外部画像URLは使わない）
- アクセシビリティに配慮（適切な見出し階層、aria 属性、alt 等）
- 1ファイルで保存できるよう、外部 CDN や外部画像ファイルは禁止
- フォントは system font stack（Web Font 不要、軽量・即時表示）

# その他

- スマホファースト設計
- リンク先（`/auth`、`/plans` 等）は仮で書いてOK（あとで実装側で差し替え）
- 既存の似たLPがあれば、敢えて差別化された雰囲気にしてほしい
- 個性的すぎず、しかし汎用SaaSすぎず、「カフェのメニュー」のような居心地良さを目指す

それでは、上記要件を満たす **完成度の高いランディングページHTML** を1ファイルで作成してください。完成後、デザインの意図と工夫した点を3〜5行で説明してください。
